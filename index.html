Extraction Modes

Three modes.
One endpoint.

Pick the mode that matches what your agent needs. Same endpoint, different optimization target.

mode: "rag"
RAG Mode
Optimized for retrieval pipelines. Content split into independently meaningful chunks, each with token counts. Ideal for vector databases and semantic search.
mode: "summary"
Summary Mode
Dense 3–5 sentence summary plus key points and entities. Perfect when you need quick context without full content — saves tokens in long agent chains.
mode: "full"
Full Mode
Everything: complete markdown, all sections, tables, code blocks, links, entities, and metadata. Use when your agent needs the complete picture.

FAQ

Common questions.

How is this different from BeautifulSoup or Firecrawl? ▼
BeautifulSoup gives you raw text — you still need to clean, structure, and format it yourself. Firecrawl is a full crawling platform built for large-scale scraping. ContextAPI does one thing: takes a single URL and returns content that's immediately usable by an LLM — no extra code on your end.
Does it work on paywalled or JavaScript-heavy sites? ▼
For public pages, yes — including most JavaScript-rendered sites. For pages behind logins or hard paywalls, the extraction will return whatever is publicly visible. Full authenticated session support is on the roadmap.
What counts as one credit? ▼
One POST to /api/extract = one credit, regardless of page size or extraction mode. Failed requests (invalid URLs, fetch errors) do not use a credit.
How fast is it? ▼
Most extractions complete in 1–2.5 seconds. This includes fetching the page, cleaning HTML, and running the Claude Haiku structuring pass. Pages that are slow to load or very long may take up to 5 seconds.
Do unused credits expire? ▼
Free tier credits don't expire. Paid plan credits roll over for 1 month (Builder) or 3 months (Scale), then reset on renewal.
